📖 História do Projeto

A Ideia

Imagine um sistema como Genshin Impact — você puxa personagens aleatórios usando "Gems" e com cada puxada há uma chance de conseguir um personagem lendário ✨

Objetivo: Construir este sistema do zero usando Java puro sem bibliotecas externas, focando em princípios POO profissionais

O Desafio

Construir um sistema completo incluindo: motor de invocação aleatória, sistema de preços, servidor TCP, cliente Swing e efeitos animados

Tudo em 47 arquivos Java em 3 camadas arquiteturais com separação total de responsabilidades

✅ Requisitos de POO

✓
Herança (Inheritance) — 3 níveis
BaseEntity → Character → Hero / Support / Antagonist
✓
Abstração — Classes Abstratas + Métodos Abstratos
abstract class BaseEntity { abstract String generateReport(); }
✓
Polimorfismo — @Override × 21 vezes
@Override public String generateReport() { ... } em Hero, Support, Antagonist
✓
Encapsulamento — 134 private + 13 protected
private String name; public String getName() { return name; }
✓
Interfaces — com Generics
interface SummonFactory<T> { T create(Rarity rarity); }

🧩 Padrões de Projeto (Design Patterns)

✓
Template Method
Character.generateReport() — esqueleto no pai, detalhes nos filhos
✓
Factory Method — 3 fábricas
CharacterFactory.create(rarity) → Hero | Support | Antagonist
✓
Chain of Responsibility
CommandRouter → SummonHandler → InventoryHandler → PlayerHandler
✓
Strategy
SummonHandler.handle() / InventoryHandler.handle() — estratégias diferentes por comando
✓
Observer (Listeners)
ActionListener nos botões de invocação → triggerAnimation() → updateUI()
